神州网信政府版WIN10设置

不错的系统

精简掉了很多平时用不到的服务，我是安装在PVE中用迅雷的，在闲置状态下CPU仅仅占用2%左右，内存仅仅用了1.5GB。

相比之下WIN10 LTSC在闲置状态下CPU占用要50%左右，内存要用3GB，主要是Software Protection服务在运行，又关不掉。

而且相比国外的Tiny10系统，核心功能很完善。Tiny10我死活无法使用远程桌面。

结论：很适合安装在PVE这类资源有限的主机上长期运行。（PS: 后来找到了Debian系的迅雷，有些资源Transmission没速度，只能靠迅雷；但不能用会员）

安装中的问题

  1. KVM配置（ID: 6028）
    1. 内存 - 4G
    2. CPU - Host
    3. BIOS - UEFI (不添加EFI硬盘)
    4. 硬盘 - SCSI
    5. 网卡 - virtio (半虚拟化)，
    6. 光驱1 - CMGE iso
    7. 光驱2 - virtio iso
  2. 在安装时倒是可以选择光驱中的驱动（virtio scsi），安装之后居然没有光驱，又没有网卡
    1. 临时方案
      1. 没什么好办法，只能先找了个备用的WINDOWS 9028，新建一块硬盘挂载上去（默认32G）
      2. 将virtio iso的内容复制进去
      3. 分离硬盘，在zfs下zfs rename rpool/data/vm-9028-disk-3 rpool/data/vm-6028-disk-3
      4. nano /etc/pve/qemu-server/6028.conf
      5. 找到scsi1: local-zfs:vm-6028-disk-0,cache=writeback,size=80G
      6. 在下面加上scsi2: local-zfs:vm-6028-disk-3,cache=writeback,size=32G
    2. 问题解决
      1. 后来发现在PVE -> Options -> Boot Order 中必须要勾选光驱
